# Sora Warehouse Management System

A **secure** real-time logistics warehouse management system for tracking nail inventory with QR code mobile scanning, PIN authentication, and Supabase cloud database.

## 🔒 Security Features

- **PIN Authentication** - 6-digit PIN required for access
- **Session Management** - Auto-logout after 1 hour of inactivity
- **Rate Limiting** - Protection against brute force attacks (5 attempts per 5 minutes)
- **Secure Sessions** - HTTPOnly cookies and CSRF protection
- **Dynamic QR Codes** - QR code updates automatically with changing IP addresses
- **Environment Security** - Sensitive credentials stored in .env file

## Features

- 📦 Real-time inventory tracking

## Installation

1. Install required packages:
```bash
pip install -r requirements.txt
```

2. Configure environment variables in `.env`:
```
SUPABASE_URL=your-supabase-url
SUPABASE_KEY=your-supabase-anon-key
SECURITY_PIN=123456
```

**Note:** If `SECURITY_PIN` is not set, a random PIN will be generated on each startup and displayed in the terminal.

3. Create database tables in Supabase:
```bash
python setup_database.py
```
Then run the displayed SQL in your Supabase SQL Editor.

## Usage

1. Start the server:
```bash
python3 processor.py
```

2. The system will display:
   - 🔒 **Security PIN** (note this down!)
   - 📱 **QR Code** for mobile access
   - 🌐 **Network information** (IP address and port)

3. Scan the QR code with your phone (must be on same WiFi network)

4. Enter the 6-digit PIN shown in the terminal

5. Use the interface to:
   - Select IN/OUT action
   - Choose nail type and enter identifier (must be X/S/C/M/A/B followed by numbers)
   - Select size and quantity
   - View all stock inventory
   - Generate reports
   - Manage product names

## Public Deployment (Access From Anywhere)

For phone access outside your local WiFi, deploy behind HTTPS.

1. Set production environment variables:
```
SUPABASE_URL=your-supabase-url
SUPABASE_KEY=your-supabase-anon-key
SECURITY_PIN=123456
SECRET_KEY=your-long-random-secret
SESSION_COOKIE_SECURE=true
PORT=8080
```

2. Start with Gunicorn (production):
```bash
gunicorn -w 2 -b 0.0.0.0:${PORT:-8080} pipeline.web.input_system:app
```

3. Deploy to any HTTPS-capable host (Render/Fly.io/Railway/VPS+Nginx)

4. Open the public URL on your phone and install it as a PWA

### Render (Ready To Deploy)

This repository includes [render.yaml](render.yaml) so you can deploy with Render Blueprint.

1. Push this project to GitHub.
2. In Render, click New + and choose Blueprint.
3. Select your repository and Render will detect [render.yaml](render.yaml).
4. Set these required env vars when prompted:
   - SUPABASE_URL
   - SUPABASE_KEY
   - SECURITY_PIN
5. Deploy.
6. Open your Render URL on phone and install it to your home screen.

Notes:
- SECRET_KEY is auto-generated by Render from [render.yaml](render.yaml).
- SESSION_COOKIE_SECURE is enabled in [render.yaml](render.yaml) for HTTPS-only cookies.
- Health check is set to /login.

## Security Best Practices

1. **Keep your .env file secure** - Never commit it to version control
2. **Set a strong PIN** - Use a memorable but secure 6-digit PIN
3. **Secure your WiFi** - Use WPA2/WPA3 encryption on your network
4. **Regular backups** - Your data is in Supabase, ensure you have backups
5. **Update the PIN** - Change your PIN periodically by updating the .env file
6. **Monitor access** - Check for unusual activity in your inventory
7. **Use HTTPS** - Required for production deployment and secure cookies

## Working in Different Environments

The system is designed to handle changing IP addresses:
- QR code automatically updates with your current local IP
- Just scan the new QR code when you change networks
- Your PIN remains the same if set in .env
- Session data is stored securely and expires after 1 hour
- For remote access from anywhere, use the deployed HTTPS URL instead of local network QR code

## How It Works

1. The system starts a local web server
2. A QR code is generated with your local IP address
3. Scan the QR code with your phone
4. Access the mobile-friendly web interface
5. All changes are saved to Supabase database in real-time
6. Each item is tracked with Type + Identifier + Size (e.g., S_S1_M)
